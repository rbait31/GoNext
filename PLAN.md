# PLAN.md
## Поэтапный план реализации приложения GoNext

---

## Этап 0. Подготовка инфраструктуры

1. Установить и настроить SQLite для React Native (expo-sqlite)
2. Создать базовую структуру папок: `lib/`, `components/`, `hooks/`
3. Настроить типы TypeScript для сущностей (Place, Trip, TripPlace)
4. Создать схему БД и скрипты миграции
5. Реализовать слой доступа к данным (data layer / DAL)

---

## Этап 1. База данных

1. Спроектировать таблицы:
   - `places` (id, name, description, visitlater, liked, lat, lng, createdAt)
   - `trips` (id, title, description, startDate, endDate, createdAt, current)
   - `trip_places` (id, tripId, placeId, order, visited, visitDate, notes)
   - `place_photos` (id, placeId, filePath)
   - `trip_place_photos` (id, tripPlaceId, filePath)
2. Реализовать инициализацию БД при первом запуске
3. Реализовать CRUD-операции для Place
4. Реализовать CRUD-операции для Trip и TripPlace

---

## Этап 2. Главный экран и навигация

1. Переработать текущий экран Home в главное меню:
   - Заголовок: GoNext
   - Кнопки: Места, Поездки, Следующее место, Настройки
2. Настроить маршруты Expo Router:
   - `/` — главное меню
   - `/places` — список мест
   - `/places/[id]` — карточка места
   - `/trips` — список поездок
   - `/trips/[id]` — детали поездки
   - `/next` — следующее место
   - `/settings` — настройки
3. Оформить навигацию компонентами React Native Paper (FAB, List, Card, etc.)

---

## Этап 3. Режим «Места»

1. Экран списка мест:
   - Загрузка мест из БД
   - Отображение в виде списка (List.Item, Card)
   - Фильтр: visitlater / liked / все
   - Кнопка добавления нового места
2. Экран создания/редактирования места:
   - Поля: name, description, visitlater, liked, координаты (dd)
   - Кнопка «Выбрать на карте» (экран карты / picker координат)
   - Прикрепление фотографий (expo-image-picker + локальное сохранение)
3. Карточка места:
   - Просмотр всех данных
   - Редактирование
   - Открытие на карте
   - Открытие в навигаторе (Linking API → Google Maps / Maps)

---

## Этап 4. Режим «Поездки»

1. Экран списка поездок:
   - Список поездок с датами
   - Выделение текущей поездки (current)
   - Кнопка создания поездки
2. Экран создания поездки:
   - Поля: title, description, startDate, endDate
   - Установка current
3. Экран деталей поездки (маршрут):
   - Упорядоченный список мест (TripPlace)
   - Drag-and-drop для изменения порядка
   - Добавление мест из базы или создание новых
   - Чекбокс visited, дата визита, заметки
   - Прикрепление фотографий к каждому пункту
4. Режимы отображения:
   - План (до поездки)
   - Дневник (после / во время поездки)

---

## Этап 5. Режим «Следующее место»

1. Экран «Следующее место»:
   - Запрос к БД: активная поездка (current = true)
   - Поиск первого TripPlace с visited = false
2. UI:
   - Карточка с информацией о месте
   - Кнопка «Открыть на карте»
   - Кнопка «Открыть в навигаторе»
3. Обработка случая: нет активной поездки или все места посещены (пустой экран с подсказкой)

---

## Этап 6. Геолокация и карты

1. Интеграция карты (expo-maps или react-native-maps)
2. Отображение места на карте
3. Получение координат через геолокацию или выбор точки на карте
4. Ссылка на внешний навигатор (Google Maps / Apple Maps через Linking)

---

## Этап 7. Фотографии

1. Съёмка/выбор фото (expo-image-picker)
2. Сохранение в файловую систему приложения (expo-file-system)
3. Привязка к Place и TripPlace
4. Отображение галереи в карточках

---

## Этап 8. Экран настроек

1. Базовые настройки (по необходимости для MVP)
2. О приложении / версия
3. Заглушка под будущие функции

---

## Этап 9. Тестирование и доработка

1. Проверка офлайн-режима (отключение сети)
2. Тестирование на Android и iOS
3. Исправление багов и улучшение UX
4. Оптимизация производительности (индексы БД, ленивая загрузка)

---

## Порядок зависимостей

```
Этап 0 (инфраструктура) → Этап 1 (БД) → Этап 2 (навигация)
                                              ↓
Этап 3 (Места) ←—————————————————————————————┘
Этап 4 (Поездки) ←— оба используют БД и навигацию
Этап 5 (Следующее место) ←— зависит от Этапов 1, 2, 4
Этап 6 (карты) — используется в Этапах 3, 5
Этап 7 (фото) — используется в Этапах 3, 4
Этап 8 (настройки) — независимый
Этап 9 — финальный
```
